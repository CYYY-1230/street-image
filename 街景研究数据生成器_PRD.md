# 街景研究数据生成器 PRD

版本：v0.1  
日期：2026-06-28  
定位：面向城市规划、风景园林、地理信息、建筑学、交通、公共健康等方向硕博论文与科研项目的街景图像采集、语义分割与指标计算工具。

## 1. 背景与机会

大量硕士、博士论文和科研项目需要使用街景图像研究城市建成环境，例如绿视率、天空开阔度、街道界面、步行空间、视觉熵、街道活力、空间品质、城市更新、健康环境暴露等。

当前用户通常需要自己完成以下流程：

- 查找或整理研究区边界。
- 获取道路网络。
- 沿路网生成采样点。
- 处理 WGS84、GCJ02、BD09 等坐标系转换。
- 调用百度地图、Google Street View 或其他街景 API 下载图像。
- 设置方向、视角、分辨率、采样间隔、年份等参数。
- 清洗无街景、下载失败、模糊、遮挡、重复图像。
- 使用 Mask2Former、DeepLabV3+、PSPNet、FCN 等模型做语义分割。
- 计算绿视率、天空开阔度、建筑占比、道路占比、视觉熵等指标。
- 导出 CSV、GeoJSON、SHP、图片、mask、论文方法说明。

这个流程对非计算机专业学生门槛较高，且容易在采样逻辑、坐标系、图像命名、失败记录、指标口径上出错。市场上已有人工代抓、代分割、按城市或按张数报价的服务，但交付方式偏非标准化、不可自助、不可复现。

本产品希望把这一流程做成可视化、自动化、可复现的数据生成工具。

## 2. 产品目标

### 2.1 一句话描述

用户在地图上选择研究区域，输入自己的地图 API Key，设置采样与分析参数，即可自动生成街景图像、语义分割结果和论文可用的空间指标数据包。

### 2.2 MVP 目标

- 支持在地图上通过矩形框或行政区选择研究范围。
- 支持导入行政区边界、路网或采样点。
- 支持沿路网按固定间隔生成采样点。
- 支持输入百度地图 API Key 获取街景图像。
- 支持四方向图像采集。
- 支持语义分割。
- 支持计算常用街景指标。
- 支持导出论文/GIS 可直接使用的数据包。

### 2.3 非目标

MVP 阶段暂不优先解决：

- 大规模全国级历史街景库存。
- 完整在线模型训练平台。
- 复杂遥感、POI、人口、房价等多源数据融合分析。
- 论文自动写作。
- 代替百度、Google 等地图平台的数据授权服务。

## 3. 目标用户

### 3.1 核心用户

1. 城市规划、风景园林、建筑学、地理信息、交通工程等专业硕博生。
2. 正在写论文，需要街景图像和绿视率、天空开阔度、街道空间指标的学生。
3. 没有稳定编程能力，但能理解 GIS、研究区、采样点、指标表等概念的用户。

### 3.2 次级用户

1. 高校教师、课题组、科研助理。
2. 城市更新、街道品质评估、景观评价相关咨询团队。
3. 需要批量街景数据的研究机构。

## 4. 用户痛点

- 不知道如何把研究区变成可采样的街景点。
- 不知道采样间隔该设置为 50m、100m 还是 200m。
- 不清楚百度地图 API 参数，如 heading、pitch、fov、coordtype、panoid。
- 坐标系混乱，WGS84、GCJ02、BD09 容易错位。
- 图片命名不规范，后续无法追踪到采样点。
- 下载失败后不知道哪些点缺失。
- 不会搭建语义分割环境。
- 不知道 ADE20K、Cityscapes、Mapillary Vistas 的区别。
- 只想要绿视率，但实际还需要 mask、类别占比、空间表格和 GIS 文件。
- 找人工服务价格不透明，交付不可控，不容易复现论文方法。

## 5. 核心场景

### 5.1 论文用户生成某区绿视率数据

用户选择“上海市徐汇区”，设置采样间隔 100m，选择四方向街景，输入百度地图 API Key，勾选“绿视率”“天空开阔度”“建筑占比”“道路占比”，提交任务。系统自动生成采样点、下载图像、做语义分割、计算指标，并导出 CSV、GeoJSON、SHP 和图片包。

### 5.2 用户已有采样点，只需要抓图

用户上传包含经纬度的 CSV 或 SHP，选择坐标系为 WGS84，系统转换为百度坐标，按每个点 0/90/180/270 四个方向抓取街景图，并输出成功图像与失败点列表。

### 5.3 用户已有街景图，只需要语义分割和指标

用户上传图片包，系统读取图片或文件名中的经纬度、方向、时间信息，选择 Mask2Former + ADE20K 或 Mask2Former + Cityscapes，输出分割 mask、类别占比和指标表。

### 5.4 用户想做历史街景对比

用户选择同一研究区，选择多个年份或指定日期，系统尝试获取不同年份街景图像，生成不同年份的绿视率和街景指标对比表。此能力作为后续版本。

## 6. 功能需求

## 6.1 项目创建

用户可以创建一个研究项目。

字段：

- 项目名称。
- 研究城市。
- 研究区名称。
- 项目用途：论文、课程作业、课题、商业项目。
- 地图数据源：百度地图、Google Street View、Mapillary、已有图片。
- 默认坐标系。

验收标准：

- 用户创建项目后进入地图工作台。
- 所有下载图像、采样点、分割结果、指标表都归属于该项目。

## 6.2 研究区选择

支持三种方式：

1. 地图矩形框选。
2. 按行政区选择：省、市、区县、街道。
3. 上传边界文件：GeoJSON、SHP、KML。

关键要求：

- 地图上显示研究区边界。
- 支持编辑边界。
- 支持显示面积、预计路网长度、预计采样点数量。
- 行政区选择后可微调边界。

验收标准：

- 用户可以用矩形框生成研究区。
- 用户可以选择至少到区县级行政区。
- 上传 GeoJSON 后可正确显示边界。

## 6.3 路网数据

支持两种方式：

1. 系统自动获取路网。
2. 用户上传路网文件。

路网字段：

- road_id
- road_name
- road_type
- geometry
- source

路网清洗：

- 可排除高速路、快速路、隧道、桥梁、封闭道路。
- 可选择只保留步行可达道路。
- 可按道路等级筛选。

验收标准：

- 系统能在研究区内生成可采样路网。
- 用户上传 SHP/GeoJSON 路网后，系统能识别线要素。
- 系统能展示路网长度和预计采样点数量。

## 6.4 采样点生成

支持沿路网按固定距离生成采样点。

参数：

- 采样间隔：25m、50m、100m、200m、自定义。
- 是否包含路段端点。
- 是否合并过近点。
- 去重点距离阈值。
- 坐标系输出：WGS84、GCJ02、BD09。

采样点字段：

- point_id
- lng
- lat
- coord_type
- road_id
- road_name
- admin_code
- admin_name
- sample_interval
- source

验收标准：

- 用户选择采样间隔后，地图实时预估采样点数量。
- 采样点可以在地图上预览。
- 用户可以下载采样点 CSV、GeoJSON、SHP。

## 6.5 街景图像获取

MVP 优先支持百度地图街景静态图。

用户输入：

- 百度地图 API Key。
- 坐标系类型。
- 图像尺寸。
- 视角方向。
- pitch。
- fov。
- 是否获取全景图。
- 是否获取四方向图。
- 是否跳过已下载图片。
- 并发数。
- 失败重试次数。

默认参数：

- 图像方向：0、90、180、270。
- pitch：0。
- fov：90。
- 采样间隔：100m。
- 图片命名：项目名_点ID_道路名_经度_纬度_方向_日期.jpg。

图像类型：

- 四方向图。
- 全景图。
- 全景图转鱼眼图。
- 历史街景图，后续版本支持。

输出：

- 原始图片。
- 下载成功清单。
- 下载失败清单。
- API 请求日志。
- 点位与图片映射表。

验收标准：

- 用户输入有效 API Key 后可以下载测试点图片。
- 下载过程中显示进度、成功数、失败数、剩余数。
- 下载失败的点会进入失败列表，并支持重新下载。
- 每张图都能追踪到 point_id、方向、坐标和道路名称。

## 6.6 图片管理

图片列表支持：

- 按行政区筛选。
- 按道路筛选。
- 按方向筛选。
- 按下载状态筛选。
- 按是否完成分割筛选。
- 按质量状态筛选。

图片预览展示：

- 原图。
- 坐标。
- 方向。
- 道路名称。
- 采样点编号。
- 下载时间。
- 对应 mask。

验收标准：

- 用户可以快速查看某个点四个方向的图像。
- 用户可以标记低质量图片。
- 用户可以批量删除或排除图片。

## 6.7 语义分割

MVP 支持预置模型推理，不做在线训练。

可选模型：

- Mask2Former + ADE20K，适合通用城市与景观类别，类别更丰富。
- Mask2Former + Cityscapes，适合道路、车辆、行人、建筑等街道交通场景。
- Mask2Former + Mapillary Vistas，适合街景场景，类别更贴近道路环境。
- DeepLabV3+，作为轻量或教学选项，后续支持。

输出：

- segmentation_mask.png
- overlay_preview.jpg
- class_pixel_count.csv
- class_ratio.csv
- image_level_metrics.csv

核心类别：

- vegetation
- sky
- road
- sidewalk
- building
- wall
- fence
- pole
- traffic_light
- traffic_sign
- person
- rider
- car
- bus
- truck
- bicycle
- motorcycle
- terrain
- water

验收标准：

- 用户可以选择模型并批量分割图片。
- 系统展示分割进度。
- 每张图片都有类别像素数和类别占比。
- mask 与原图可以叠加预览。

## 6.8 指标计算

MVP 指标：

- 绿视率 GVI：植被像素 / 有效像素。
- 蓝视率 BVI：天空或水体像素 / 有效像素。
- 天空开阔度：天空像素 / 有效像素。
- 建筑占比：建筑像素 / 有效像素。
- 道路占比：道路像素 / 有效像素。
- 人行空间占比：人行道像素 / 有效像素。
- 车行空间占比：道路 + 车辆相关像素 / 有效像素。
- 硬质铺装占比：道路 + 人行道 + 地面硬质类别 / 有效像素。
- 人车可见密度：人、车、骑行者等类别占比。
- 视觉熵：基于类别占比计算视觉复杂度。
- 自然度：植被、天空、水体、地形等自然类元素占比。
- 界面围合度：建筑、墙、栅栏等垂直界面元素占比。

色彩统计指标：

- RGB 均值。
- RGB 方差。
- LAB 均值。
- HSV 色相、饱和度、亮度均值。
- 色彩丰富度 CVI。

聚合尺度：

- 图片级。
- 点位级，四方向平均。
- 路段级。
- 网格级。
- 行政区级。

验收标准：

- 用户可以勾选需要的指标。
- 系统输出每张图片、每个点、每条路段的指标表。
- 指标口径在导出包中自动生成说明文件。

## 6.9 结果导出

导出格式：

- ZIP 数据包。
- CSV。
- GeoJSON。
- SHP。
- 图片文件夹。
- mask 文件夹。
- PDF 或 Markdown 方法说明。

标准导出目录：

```text
project_name/
  01_boundary/
  02_road_network/
  03_sample_points/
  04_streetview_images/
  05_segmentation_masks/
  06_metrics/
  07_failed_records/
  08_method_description/
```

核心表格：

- sample_points.csv
- image_manifest.csv
- failed_downloads.csv
- segmentation_class_ratio.csv
- image_metrics.csv
- point_metrics.csv
- road_metrics.csv
- admin_metrics.csv

验收标准：

- 用户可以下载完整 ZIP。
- CSV 可直接用 Excel 打开。
- GeoJSON/SHP 可直接导入 QGIS 或 ArcGIS。
- 方法说明包含采样间隔、方向、模型、数据源、坐标系、指标公式。

## 6.10 任务与计费预估

在用户正式运行任务前，系统展示预估：

- 研究区面积。
- 路网长度。
- 采样点数量。
- 图片数量。
- API 调用次数。
- 预计耗时。
- 预计本地/云端计算资源。
- 可能费用。

计算方式示例：

- 图片数量 = 采样点数量 x 方向数量 x 年份数量。
- 四方向默认方向数量为 4。
- 全景图默认方向数量为 1。
- 语义分割费用按图片数量、模型、分辨率估算。

验收标准：

- 用户提交任务前能看到明确数量预估。
- 用户可以通过调整采样间隔、方向数量、研究区范围降低任务规模。

## 10. 权限与合规

### 10.1 API Key

- 用户可以使用自己的百度地图 API Key。
- 系统不应明文存储用户 API Key。
- API Key 需要加密保存或仅在本地任务中临时使用。
- 用户需要自行确认地图平台的调用额度、费用和使用条款。

### 10.2 数据使用

- 产品需要提示用户遵守百度地图、Google Street View、Mapillary 等平台的数据使用协议。
- 若提供云端代下载，需要明确数据来源、授权方式和可用范围。
- 若只是本地工具调用用户自己的 Key，需要在产品说明中明确责任边界。

### 10.3 隐私处理

后续可提供：

- 人脸模糊。
- 车牌模糊。
- 原图仅本地保存。
- 云端处理后自动删除。

## 13. 竞品与市场观察

从现有人工服务可以看到明确需求：

- 按城市、区县、采样点数量或图片数量报价。
- 常见采样间隔为 50m、100m、200m。
- 用户关心四方向图、全景图、历史街景。
- 交付内容包括街景图、语义分割结果、类别占比和指标计算。
- 常用模型包括 Mask2Former、DeepLabV3+ 等。
- 常见标签体系包括 ADE20K、Cityscapes、Mapillary Vistas。
- 常见指标包括绿视率、天空开阔度、蓝视率、建筑占比、道路占比、人行空间占比、车辆密度、视觉熵、自然度、景观多样性。
- 用户希望图片文件名包含日期、经纬度、道路名，方便查看和复核。

产品机会在于把人工服务标准化、自助化、可复现化，并降低用户理解 API、坐标系、模型环境和 GIS 导出的门槛。

## 14. 风险与待验证问题

### 14.1 数据源风险

- 百度街景 API 是否支持用户需要的全部历史年份。
- 不同城市街景覆盖率差异较大。
- API 调用限制可能影响大规模下载。
- 地图平台条款可能限制数据存储、转售或公开发布。

### 14.2 技术风险

- 大规模图片下载可能需要任务队列和断点续传。
- 语义分割需要 GPU，云端成本需要控制。
- 不同模型标签体系不同，指标映射需要标准化。
- 坐标系转换错误会严重影响数据可信度。

### 14.3 产品风险

- 用户需求差异大，可能有人只要图片，有人只要分割，有人要完整论文数据。
- 过多参数会吓退非技术用户。
- 需要在“专业可配置”和“傻瓜式运行”之间取得平衡。
