# StreetScope 上线方案：便宜网站常开 + GPU 按需开

## 推荐架构

不要把网站放在 GPU 云电脑上。GPU 太贵，而且你不需要 24 小时开着。

推荐拆成两台机器：

1. Web 服务器：一直开，便宜配置即可。
   - 用途：登录、网页、历史项目、任务记录、街景图片归档、最终 ZIP。
   - 推荐配置：1 核 1G 或 2 核 2G，系统盘 40-80G 起步。
   - 常开成本低。

2. GPU 服务器：只在做语义分割时开。
   - 用途：Mask2Former / FCN / PSPNet / DeepLabv3 推理。
   - 开机后在网页“模型服务地址”填：`http://GPU服务器IP:9000/segment`
   - 用完关机，省钱。

## 你需要提供给我的东西

### 必需

- 一台便宜 Web 服务器的 SSH 信息：
  - IP
  - SSH 端口
  - 用户名，通常是 `root`
  - 密码或密钥

- 你想设置的网站登录账号和密码：
  - 账号，例如 `admin`
  - 密码，建议 12 位以上

### 强烈建议

- 一个域名，例如 `streetscope.xxx.com`
  - 没域名也能先用 `http://服务器IP` 访问。
  - 有域名后，可以自动 HTTPS，更像正式产品。

### 已经有

- GPU 服务器已经配置好模型服务：
  - `http://117.50.216.65:9000/segment`
  - 这台不用一直开，只在语义分割时开。

## 第一次部署 Web 服务器

在本地项目根目录执行：

```bash
cp deploy/env.example deploy/.env
```

打开 `deploy/.env`，改这些：

```bash
SERVER_HOST=你的Web服务器IP
SERVER_USER=root
SSH_PORT=22
DOMAIN=你的域名，没有就留空
WEB_USER=admin
WEB_PASSWORD=你的网站登录密码
APP_DIR=/opt/streetscope
DATA_DIR=/var/lib/streetscope
DEFAULT_SEGMENTATION_URL=
```

然后执行：

```bash
set -a
source deploy/.env
set +a
bash deploy/push_update.sh
```

完成后：

- 有域名：打开 `https://你的域名`
- 没域名：打开 `http://你的Web服务器IP`

浏览器会弹出账号密码框，输入 `WEB_USER` / `WEB_PASSWORD`。

## 平时怎么用

1. 只做研究区、路网、采样、街景下载：
   - 只需要 Web 服务器开着。
   - GPU 可以关机。

2. 要做语义分割：
   - 打开 GPU 服务器。
   - 确认模型服务通：

```text
http://GPU服务器IP:9000/health
```

   - 在网页“模型服务地址”填：

```text
http://GPU服务器IP:9000/segment
```

   - 点“用已下载图片计算指标”。

3. 分割完成并导出 ZIP 后：
   - 可以关掉 GPU 服务器。
   - 历史和 ZIP 仍在 Web 服务器。

## 本地继续 vibe coding 后怎么更新线上

我在本地帮你改完代码后，你只需要执行：

```bash
set -a
source deploy/.env
set +a
bash deploy/push_update.sh
```

这个脚本会自动：

- 同步代码到 Web 服务器
- 安装/更新后端依赖
- 重新构建前端
- 重启 Web 服务
- 保留 `/var/lib/streetscope` 里的历史数据

## 数据放在哪里

线上 Web 服务器：

```text
/var/lib/streetscope
```

这里保存：

- 历史任务
- 街景图片归档
- 分割结果
- 导出 ZIP 依赖的文件

不要把这个目录删掉。
